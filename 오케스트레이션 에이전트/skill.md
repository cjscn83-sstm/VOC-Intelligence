# Skill: VOC 오케스트레이션 (조율·종합·렌더링)

오케스트레이션 에이전트가 파이프라인 전체를 관리할 때 사용하는 스킬.

## 언제 사용하나
- 여러 분석 에이전트의 독립 의견을 하나의 일관된 보고서로 합쳐야 할 때
- 통계 대시보드 + 에이전트 의견 + 개선 로드맵을 단일 산출물로 렌더링할 때

## 절차
1. **입력 정규화** — `25년 VOC원문.xlsx`(788건) 로딩, 스키마·결측 검증, 공유 데이터셋 준비
2. **서브에이전트 격리 실행**
   - 현황분석·웹리서치는 **독립(병렬)** 으로 실행 (상호 결과 미노출)
   - 개선제안은 현황분석 근거를 입력받아 **후행** 실행
3. **결과 수집** — 각 에이전트의 구조화 의견(headline + 섹션별 title/summary) 취합
4. **교차 검증·종합** — 상충 결론은 근거 강도로 판정, 통합 headline·총평 도출
5. **렌더링** — 통계 차트(월별 추이·심각도·유형·지사) + 3개 에이전트 카드 + 카테고리 현황 + 개선 로드맵 + 총평을 단일 HTML로 생성
6. **배포** — GitHub Pages 공개 URL 발행 및 공유

## 종합 규칙
- 데이터 근거 > 인상·직관
- 표면 증상 → 근본 원인으로 재분류 (예: 불친절 → 불완전판매)
- 모든 개선 결론은 담당·기한·KPI가 있는 실행 과제로 귀결
