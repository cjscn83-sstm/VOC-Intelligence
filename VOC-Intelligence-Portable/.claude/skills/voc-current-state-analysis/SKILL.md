# Skill: VOC 현황 분석 (분류·집계·리스크 스코어링)

## 언제 사용하나
- 원본 VOC 엑셀을 정량·정성 분석해 현황을 진단할 때
- 고위험 사례와 조직/시계열 패턴을 데이터로 뽑을 때

## 절차
1. **로딩·검증** — xlsx 로딩, 결측·중복·인코딩 정합성 확인
2. **유형 분류(라벨링)** — 규칙/키워드 + LLM 보조로 카테고리 지정 (불완전판매, 끼워팔기, 설치 품질, 캐시백/정산 등)
3. **귀책 판정** — 각 건의 근본 원인 소재를 C/A/P/X로 판정 (아래 표) + 원문 근거 인용
4. **스코어링** — 심각도(경/중/고), 감정 격화 스코어 → 고위험(예: 86건) 식별. 격화 요인(사후 대응 실패)은 최초 원인과 분리
5. **집계** — 유형 × 지사 × 월 + **유형 × 귀책** 피벗, 이상치·편중 탐지
6. **사례 추출** — 대표 고위험 사례 상위 5건 심층 리뷰
7. **한계 명시** — 표본·누락·분류 모호성 유의점 기록

## 귀책(책임 소재) 분류 코드
| 코드 | 귀책 | 정의 | 판정 신호 |
|------|------|------|-----------|
| **C** | 고객 (억지·무리) | 계약·정책·상식 밖 과도/반복 요구, 허위 주장 | 규정 밖 보상 반복 요구, 근거 없는 환불, 폭언 |
| **A** | 상담사 (응대) | 개인 태도·지식·실수 | 불친절, 오안내, 약속 미이행, 감정적 응대 |
| **P** | 회사·프로세스 (제도·시스템) | 규정 미비·모순, 시스템 오류, 핸드오프 실패 | 표준 절차 부재, 정산 오류, 부서 떠넘기기 |
| **X** | 복합·불명 | 요인이 얽혀 있거나 자료로 단정 불가 | 초기 P가 A로 격화 등 |

**판정 원칙**: 접수 유형이 아니라 **근본 원인**으로 분류. "불친절(A)"이라도 원인이 불완전판매면 **P로 재분류**하고 근거를 남긴다. 이 귀책 코드는 개선제안 에이전트의 워크플로우 선택 입력이 된다.

## 분석 축
유형별 · **귀책(책임 소재)** · 감정·리스크 · 대표 사례 · 시계열·조직 · 데이터 한계

## 출력 형식
headline + 섹션별 {title, summary} + **유형×귀책 집계표** (모든 판정에 근거 인용 포함)
