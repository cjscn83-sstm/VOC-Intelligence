# Skill: VOC 개선 제안 (실행 과제 설계·우선순위화·로드맵)

## 언제 사용하나
- 분석 결과를 담당·기한·지표가 붙은 실행 과제로 바꿔야 할 때
- 단계별 개선 로드맵과 조직별 적용안을 설계할 때

## 절차
1. **레버 매핑** — 현황분석 **귀책(C/A/P/X)·유형**별 개선 레버 도출
2. **워크플로우 선택** — 귀책·상황에 맞는 4대 패턴(순차/병렬/분기/순환) 중 주 워크플로우 지정 + 근거·출처 명시
3. **우선순위화** — 임팩트 × 실행 난이도 정렬
4. **과제화** — 각 과제에 담당·기한·KPI + 워크플로우 다이어그램 부여
5. **로드맵 배치** — 즉시 / 단기 / 중기 / 중장기 시퀀싱
6. **조직별 적용** — 지사·팀 단위 우선 적용안 작성

## 귀책별 워크플로우 매핑
| 워크플로우 | 정의 | 적합한 귀책·상황 |
|-----------|------|------------------|
| **순차 (Sequential)** | 정해진 순서로 단계 진행 | **P** 프로세스 재설계·표준 절차 배포 |
| **병렬 (Parallel)** | 동시 수행 후 합류 | **X/복합**·피크 다조직 동시 대응 |
| **분기 (Conditional)** | 조건 판정 후 경로 분기 | **C** 고객 억지: 규정 판정→응대/거절/에스컬레이션 |
| **순환 (Loop)** | 실행→피드백→개선 반복 | **A** 상담사: 교육→모니터링→피드백 / 폐쇄루프 |

## 출처(근거) 규칙 — 필수
- **내부**: 현황분석의 귀책 판정·집계 수치 인용
- **외부**: 실제 문헌·URL 표기
  - *Workflow Patterns* (van der Aalst et al., workflowpatterns.com) — Sequence / Parallel Split / Exclusive Choice / Structured Loop
  - NPS *The Inner Loop* / *The Outer Loop* (Bain & Company)
  - 단통법 폐지·취약고객 보호 관련 공개 자료
- 불확실 시 "추정" 표기 + 검증 과제로 분리

## 5대 현장 실행 과제
1. 개통·계약 전 '3대 고지' 서명 체크리스트
2. 고위험 VOC 24시간 에스컬레이션 SLA
3. 취약고객(고령·장애) 판매 프로토콜
4. 출시·성수기 캘린더 연동 선제 대응
5. 캐시백·정산 서면 표준화 + 철회 기준 정렬

## 우선순위 근거
- 근본 원인은 불완전판매(불친절 아님)
- 격화의 진범은 사후 대응 실패 → 24시간 SLA
- 제도 변화 대응 (단통법 폐지 + 취약고객)
- 예측 가능한 피크는 선제 대응 (출시·성수기 캘린더)

## 출력 형식
5대 과제 + 실행 디테일 + 조직별 우선 적용 + 단계별 로드맵
