VOC 인텔리전스 · 분석 방법론

귀책 분류 → 워크플로우 매핑 방법론

현황분석 에이전트가 VOC의 책임 소재(귀책)를 판정하고, 개선제안 에이전트가 귀책별로 4대 워크플로우(순차·병렬·분기·순환)를 선택해 근거·출처와 함께 개선안을 제안하는 체계입니다. 2025년 삼성스토어 VOC 788건 분석 파이프라인 기준.

1 분석 파이프라인

오케스트레이션 에이전트가 3개 서브에이전트를 조율합니다. 이번 업데이트는 ②·③에 적용됩니다.

① 현황분석 (유형+귀책 판정) → ② 웹 리서치 (외부 근거) → ③ 개선제안 (워크플로우 선택+출처) → ④ 종합·대시보드

2 귀책(책임 소재) 분류 — 현황분석

표면 증상이 아니라 근본 원인의 소재로 분류합니다. 접수 유형이 '불친절(A)'이어도 원인이 불완전판매면 P로 재분류하고 원문 근거를 남깁니다. 복수 요인은 주(主)·부(副) 귀책으로 표기합니다.

C 고객 귀책 · 억지·무리한 요구

계약·정책·상식 범위를 벗어난 과도/반복 요구, 사실과 다른 주장.
신호: 규정 밖 보상 반복 요구, 근거 없는 환불·교환, 폭언·협박

A 상담사 귀책 · 응대 문제

개인의 태도·지식·실수.
신호: 불친절·무성의, 오안내, 약속 미이행, 감정적 응대

P 회사·프로세스 귀책 · 제도·시스템

규정 미비·모순, 시스템 오류, 조직 간 핸드오프 실패.
신호: 표준 절차 부재, 정산 오류, 부서 떠넘기기, 설치 품질 게이트 부재

X 복합·불명

요인이 얽혀 있거나 자료로 단정 불가.
예: 초기 P였으나 사후 대응 실패(A)로 격화 — 최초 원인과 격화 요인을 분리 기록

3 귀책별 워크플로우 선택 — 개선제안

귀책 코드를 입력받아 표준 제어흐름 패턴(순차·병렬·분기·순환) 중 주 워크플로우를 지정합니다. 하위 단계에 다른 패턴을 조합할 수 있습니다.

🔀 분기 (Conditional) → C

조건 판정 후 경로 분기. 고객 억지에 대한 규정 기준 판정.

접수→규정 판정 →응대 / 정중 거절 / 에스컬레이션
예: 규정 밖 보상 요구 표준 대응 트리

🔁 순환 (Loop) → A

실행→피드백→개선 반복. 상담사 역량·태도 개선 루프.

교육→모니터링 →피드백↺
예: 폐쇄루프(이너·아우터 루프) 코칭

➡️ 순차 (Sequential) → P

정해진 순서로 단계 진행. 프로세스 재설계·표준 절차 배포.

3대 고지→서명 →개통
예: 개통·계약 전 서명 체크리스트

⑂ 병렬 (Parallel) → X

여러 작업 동시 수행 후 합류. 복합·피크 다조직 동시 대응.

설치 품질 점검정산 대사 →합류
예: 출시·성수기 선제 대응

4 출처(근거) 규칙 — 필수

모든 워크플로우 선택과 개선 제안에는 내부 근거(귀책 판정·집계 수치)와 외부 출처(실제 문헌·URL)를 함께 명시합니다. 불확실하면 단정하지 않고 "추정"으로 표기 후 검증 과제로 분리합니다.
