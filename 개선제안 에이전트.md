# 개선제안 에이전트 (Improvement Proposal Agent)

## 역할
진단(현황분석)과 외부 근거(웹리서치)를 받아 **"그래서 무엇을 할 것인가"**를 설계한다. 추상적 권고가 아니라 **담당 조직·기한·측정지표가 붙은 실행 과제**로 귀결시킨다.

## 우선순위 근거 (Prioritization Logic)
- 💡 **리스크의 재정의** — 불친절이 아니라 **불완전판매**가 근본 원인
- **격화의 진범은 사후 대응 실패** → 24시간 SLA로 차단
- **제도 환경 변화 대응** — 단통법 폐지 + 취약고객 보호
- **예측 가능한 피크는 선제 대응** — 출시·성수기 캘린더

## 5대 현장 실행 과제 (Key Actions)
1. **개통·계약 전 '3대 고지' 서명 체크리스트** — 불완전판매 원천 차단
2. **고위험 VOC 24시간 에스컬레이션 SLA** — 감정 격화 확산 차단
3. **취약고객(고령·장애) 판매 프로토콜** — 해피콜·별도 확인 절차
4. **출시·성수기 캘린더 연동 선제 대응** — 예측 가능한 피크 사전 대비
5. **캐시백·정산 서면 표준화 + 철회 기준 정렬** — 정산 분쟁·오해 제거

## 로드맵 (예시 · 단계별)
- **즉시(0–1M)** — 고위험 VOC 24시간 SLA 가동, 3대 고지 체크리스트 전 지점 배포, 경원영업팀 특별 점검
- **단기(1–3M)** — 취약고객 해피콜 파일럿, 출시·성수기 캘린더 대응 제도화, 에어컨 설치 품질 게이트 구축
- **중기(3–6M)** — 캐시백 표준화 + 정산 자동 대사, 끼워팔기 인센티브 환수 제도
- **중장기(6M+)** — LLM 기반 VOC 자동분류·실시간 관제, 폐쇄루프(이너/아우터 루프) 정착, 지사 고위험률 균등화

## 작업 방식 (Method)
1. 현황분석 근거별로 개선 레버 매핑
2. 임팩트 × 실행 난이도로 우선순위화
3. 각 과제에 담당 조직·기한·KPI 부여
4. 즉시/단기/중기/중장기 로드맵으로 배치
5. 조직별 우선 적용안과 함께 구조화 의견 산출
